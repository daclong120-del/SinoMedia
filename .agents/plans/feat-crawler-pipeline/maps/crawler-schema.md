# 🌳 Detail Map — crawler-schema

- Thiết kế database và RLS cho dữ liệu cào
  - ├ Bảng `crawled_authors` (Tác giả) ⏳
    - ├ Unique constraint: `unique (platform, platform_uid)` ⏳
      - └ à, nghĩa là crawler có thể dùng `ON CONFLICT (platform, platform_uid) DO UPDATE` để cập nhật thông tin tác giả (avatar, nickname) mà không bị lỗi trùng lặp.
    - ├ Trường `avatar_url` ⏳
      - └ à, nghĩa là crawler cần tải avatar của tác giả lên R2 và lưu link R2 vào đây, tránh việc link gốc của nền tảng (Douyin/TikTok/XHS) bị hết hạn hoặc lỗi 403 hotlinking.
  - ├ Bảng `crawled_posts` (Bài viết / Video) ⏳
    - ├ Unique constraint: `unique (platform, platform_id)` ⏳
      - └ à, nghĩa là khi cào lại hoặc cập nhật tương tác (views, likes), crawler sẽ upsert mà không tạo bản ghi mới trùng lặp.
    - ├ Khóa ngoại `author_id` ⏳
      - └ à, nghĩa là trỏ tới bảng `crawled_authors(id)`. Khi xóa tác giả, post sẽ set null (`on delete set null`) để giữ lại dữ liệu bài viết.
    - ├ Trường `media_urls` kiểu `text[]` ⏳
      - └ à, nghĩa là lưu danh sách link media (video, ảnh) đã tải lên R2. Expo client sẽ đọc trực tiếp mảng này.
    - ├ Index `idx_crawled_posts_platform` trên `(platform, crawled_at desc)` ⏳
      - └ à, nghĩa là tối ưu cho việc truy vấn feed của Expo client (lọc theo platform và sắp xếp thời gian cào mới nhất).
  - ├ Row Level Security (RLS) ⏳
    - ├ Bật RLS cho cả 2 bảng ⏳
      - └ à, nghĩa là mặc định mọi truy cập qua API (anon) sẽ bị chặn cho đến khi được khai báo policy rõ ràng.
    - ├ Policy select cho phép Anon đọc (`select using (true)`) ⏳
      - └ à, nghĩa là ứng dụng di động hiển thị danh sách công khai, không yêu cầu user đăng nhập mới xem được.
    - ├ Không cấu hình ghi cho Anon ⏳
      - └ à, nghĩa là client-side và các bên ngoài không thể can thiệp sửa đổi dữ liệu.
      - └ à, nghĩa là crawler bắt buộc phải dùng `service_role` key để bypass RLS ghi dữ liệu vào database.
